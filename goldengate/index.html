🗺 Project Overview

What is GNO/ME?

GNO/ME (GNO Multitasking Environment) is a complete Unix-like operating system for the Apple IIgs, originally written by Devin Reade (~1997–1999). It includes a real kernel, POSIX libc, ~100 utilities, and networking — all native 65816 code.

Repo: github.com/ksherlock/gno

Goal

Build GNO/ME from source on macOS using the GoldenGate cross-compilation environment, producing a distributable .shk (ShrinkIt) archive or ProDOS disk image runnable on real Apple IIgs hardware.

The build process does not need to run on the IIgs itself.

What's in the Repo

634 C files, 88 assembly files, 101 Rez resource files, 131 Makefiles. Includes: kernel, libc (20 subdirs), 10+ libraries (curses, termcap, crypt, netdb…), ~100 utilities, shell (gsh), and full man page set.

Languages: C (~65%), 65816 Assembly (~20%), Rez resources (~15%)

Apple IIgs Architecture

The IIgs uses a 65816 processor (16-bit extension of the 6502) running at ~2.8 MHz with up to 8MB RAM. GS/OS is the native operating system. GNO/ME runs as a GS/OS application that provides a Unix-like multitasking environment on top of GS/OS.

CPU: WDC 65C816 @ 2.8 MHz · RAM: 256KB–8MB · ProDOS/HFS storage

Build Approach

The original build system uses dmake with GNO namespace paths — neither available in GoldenGate. Instead we use GNU make + iix, the same pattern proven in the tubesnob-applespi project. All new build scripts live in goldengate/.

Reference: /Users/smentzer/source/tubesnob-applespi

Distribution Format

GNO/ME was originally distributed as ~16 ShrinkIt .shk archives plus a boot disk. For this build, the /dist hierarchy (from build release targets) will be archived using nulib2 on macOS. Alternatively, a ProDOS .po/.2mg disk image using cadius.

Install: brew install nulib2 cadius

How iix Compilation Works

# Compile a C file → produces foo.a (object), foo.root, foo.sym
iix compile foo.c

# Compile with explicit output name and flags
iix compile -o foo.a foo.c

# Assemble a .asm file
iix assemble foo.asm

# Link objects into executable
iix link -o foo foo         # (note: no .a suffix on input)

# Create a library from objects
iix makelib libfoo.o +file1.a +file2.a

# Key flags:
#   +O  optimize
#   -P  suppress progress messages
#   -I  don't generate .sym file
#   cc=-DFOO  define preprocessor macro

Note on /src Namespace

The GNO /etc/namespace file maps /src → :gnosrc:src (a ProDOS volume). This is a GNO runtime concept; iix does not implement GNO namespace resolution. All build scripts must use macOS absolute paths. The original paths.mk / prog.mk / lib.mk files are preserved as reference documentation but cannot be used directly.

🏗 Build Plan

Sequence from NOTES/devel/doing.builds, adapted for GNU make + iix. Each phase depends on all previous phases completing successfully.

  • 0

    Environment Setup

    Run bash goldengate/setup.sh. Verifies iix, GoldenGate, creates output directories. Install nulib2 and/or cadius for final packaging.

  • 1

    Header Installation

    Copy all GNO headers (include/, ainclude/, rinclude/, orcacdefs/) into GoldenGate's ~/Library/GoldenGate/usr/include/ with correct ProDOS file types set via iix /bin/chtyp. Currently partially done — ainclude/ and orcacdefs/ need updating.

  • 2

    ORCALib (65816 assembly runtime)

    Source: lib/ORCALib/. Assemble 13 .asm files using iix assemble, archive with iix makelib. Original build script (lib/ORCALib/make) is in ORCA shell syntax — must be converted to GNU Makefile. Output: orcalib (ORCA library archive).

    Files: vars.asm cc.asm ctype.asm fcntl.asm orca.asm setjmp.asm signal.asm stdio.asm stdlib.asm string.asm time.asm toolglue.asm assert.asm

  • 3

    SysFloat + SysFPEFloat

    SANE (Standard Apple Numerics Environment) floating point libraries. These may come from the ORCA 2.2.0 distribution rather than being built from source. Check /Volumes/Storage/IIgs/Development/ORCA/Software/ORCAC.220B3.shk.

  • 4

    lsaneglue

    Source: lib/lsaneglue/. Thin glue layer between GNO and SANE. Simple single-directory build.

  • 5

    libc (most complex — 9 subdirs + top level)

    Each subdir must be built independently (dmake can't recurse). Build in order:

    lib/libc/gen/    lib/libc/gno/    lib/libc/locale/
lib/libc/stdio/  lib/libc/stdlib/ lib/libc/stdtime/
lib/libc/string/ lib/libc/sys/
lib/libc/        ← top level LAST (for resource fork)

    Special: stdio/vfprintf.c needs cpp preprocessing workaround. gno/ uses macgen. Top level steals assert.o from ORCALib obj tree.

  • 6

    Support Libraries

    In dependency order: libcrypt → libutil → libtermcap → libcurses → liby → netdb → libcontrib

    Skip: libedit and libsim (explicitly "not yet building" in original docs)

  • 7

    Bootstrap: catrez

    Source: usr.orca.bin/catrez/. Build before utilities since all executables need it to attach resource forks. Compile using iix compile + iix link directly.

  • 8

    Utilities

    Sources: bin/ (44 utils), usr.bin/ (52 utils), usr.orca.bin/, sbin/ (5 utils), usr.sbin/ (5 daemons). Order within this group doesn't matter.

    Special cases: bin/gsh (17 .asm files, links libtermcap.204), sbin/init (BSD make syntax — needs conversion), usr.bin/awk (needs STACK=4096, pre-generated parser files)

  • 9

    Kernel

    Sources: kern/gno/ (C + 65816 assembly), kern/drivers/. Original build documentation is incomplete ("need to add text"). Must read kern/gno/Makefile directly. Output: S16 file type (GS/OS system executable).

  • 10

    Packaging

    After all release targets, the /dist tree is the complete installable GNO/ME system. Archive with nulib2 into .shk or use cadius to produce a .po/.2mg ProDOS disk image. Skip original fudgeinstall script (requires dead v2.0.4 server).

📖 Key Repository Documentation

NOTES/devel/doing.builds

Authoritative build sequence. Written by Devin Reade. Describes the exact order in which components must be built. This is the primary reference for Phase ordering.

Last updated: 1999

NOTES/status.bin

Per-utility build and test status table. Shows which utilities were compiled (C), tested (T), or still in progress as of the original authors' last work. Use to prioritize what to attempt first.

NOTES/status.lib

Per-function status for libc. Shows which functions were reviewed (R), compiled (C), or tested (T). Useful when debugging libc build failures.

build.tools/dmake.startup

The dmake inference rules file. Defines the full toolchain (CC, LD, AR, SHELL, flags). Essential reference for translating to GNU make. Shows how occ flags map to ORCA behavior.

lib/ORCALib/make

ORCA shell script that builds ORCALib. Written in GNO/ORCA shell syntax. Must be converted to a GNU Makefile. Shows the exact assembly order and makelib invocations.

NOTES/README.src / README.beta

Notes on the source distribution, known issues, and beta status. Important warnings: libc and ORCALib must be compiled together (struct FILE coupling). Notes on broken nroff.
